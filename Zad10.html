10. Budowa selektora

1. Wymień oraz opisz dwie części selektora

Część identyfikująca – jest to część, która wskazuje, do jakiego elementu HTML odnosi się dany selektor. Może to być np. nazwa tagu HTML (np. div, p), klasa (np. .nazwa-klasy), identyfikator (np. #nazwa-id), a także bardziej zaawansowane kombinacje, jak selektory atrybutów czy pseudoklasy.

Część deklaratywna – to zbiór par "właściwość: wartość", które określają, jak dany element ma być stylizowany. Przykładem może być zapis color: red; lub font-size: 16px;. Zawiera ona zasady stylizacji dla danego elementu.

2. Znaczenie nawiasów klamrowych {...}

Nawiasy klamrowe {} w selektorze CSS służą do zamknięcia części deklaracyjnej. Wszystkie właściwości i wartości, które mają zostać przypisane do elementu, muszą być zawarte wewnątrz tych nawiasów. Na przykład:

div {
  color: blue;
  font-size: 14px;
}

W tym przykładzie, nawiasy klamrowe obejmują właściwości color i font-size, które będą zastosowane do wszystkich elementów div.

3. Cechy selektora

Cechy selektora to właściwości, które pomagają w dokładnym określeniu, jakie elementy HTML mają być stylizowane. Mogą to być:

  • Uniwersalne – np. selektor typu, który wybiera wszystkie elementy danego typu.
  • Identyfikatory – wybierają elementy o danym identyfikatorze (np. #nazwa-id).
  • Klasy – wybierają elementy, które mają przypisaną daną klasę (np. .nazwa-klasy).
  • Atrybuty – selektory atrybutów, które wybierają elementy na podstawie wartości atrybutu (np. [type="text"]).

4. Wartości w selektorze

Wartości przypisywane w selektorach określają wygląd elementów HTML. Mogą to być różne typy danych:

  • Kolory – np. color: red;, background-color: #fff;
  • Jednostki miary – np. width: 100px;, height: 50%
  • Tekst – np. font-family: Arial;, text-align: center;
  • Czas – np. transition: 0.5s ease;

Wartości mogą być liczbami, słowami kluczowymi (jak center, auto), kolorami (np. blue, #ff0000) oraz jednostkami (np. px, em).

5. Kiedy stosujemy znak średnika (;), a kiedy nie jest konieczny

Znak średnika ; służy do oddzielania kolejnych właściwości w deklaracji CSS. W przypadku wielu właściwości każda musi kończyć się średnikiem. Jednak średnik nie jest wymagany po ostatniej właściwości w danej deklaracji, chociaż zaleca się jego stosowanie, aby uniknąć błędów w przyszłości, szczególnie w przypadku edytowania kodu.

Przykład:

div {
  color: red;
  font-size: 16px;
}

W przypadku tylko jednej właściwości średnik po niej nie jest konieczny:

div {
  color: red
}
